# 產業別視角的均線交叉策略適用性檢證（論文草案規劃）

## 摘要
在 `strategies/ma-cross` 產出一份可直接用於撰寫/投稿的論文草案（Markdown），研究「均線交叉（做多/空手）策略在不同產業類別的適用性差異」，並設計嚴謹的 **產業別參數最佳化 + 樣本外（OOS）檢證** 流程。主結論以「各產業中 **有多少股票在 OOS 打敗 buy&hold**」作為適用性指標，並以統計檢定與多重比較校正支撐推論。

## 固定前提（已由你選定）
- 市場範圍：台股全市場（上市 + 上櫃）
- 產業分類：交易所官方產業別
- 策略：SMA 均線交叉，做多/空手；訊號 `position = signal.shift(1)` 避免 lookahead；報酬 close-to-close；成本以固定 `fee_bps` 表示
- 參數選擇：每個產業各自最佳化（選一組 (short,long) 套用該產業所有股票）
- 最佳化目標：最大化 Sharpe（訓練期）
- 適用性彙整主指標：勝率/超額比例（OOS 打敗 buy&hold 的比例）
- 樣本期間：固定區間（預設採 `2010-01-01` 到 `2025-12-31`，寫死於論文）

## 論文內容（`THESIS.md` 章節與必寫要點）
1. 題目與摘要
   - 中文題目（含「產業別」「均線交叉」「樣本外檢證」關鍵字）
   - 摘要：一句話研究缺口 + 方法（產業別最佳化 + OOS）+ 主發現形式（產業 outperformance 比例差異）+ 貢獻
2. 研究動機與問題定義
   - RQ1：均線交叉的 OOS 適用性是否因產業而異？
   - RQ2：產業最佳化參數 (short,long) 是否呈現系統性差異？
   - RQ3：在控制波動/趨勢性等特徵後，產業效果是否仍存在？
3. 文獻回顧（段落模板）
   - 技術分析與趨勢追蹤（MA cross）經驗研究
   - 產業異質性（風險、景氣循環、資訊擴散速度）與策略有效性
   - data-snooping / multiple testing 對策略研究的影響與常見處理
4. 資料與樣本
   - 價格：FinMind 日頻（優先用調整後價格表 `price_adj_daily`），排除 `is_placeholder=1`
   - 產業分類：官方產業別，分類時間點固定為 `2019-01-01`（OOS 起點）以避免前視偏誤；若無歷史分類則列為限制
   - 納入條件（決策完成，寫死於論文）：
     - 訓練期有效交易日數 `>= 504`（約 2 年）
     - 測試期有效交易日數 `>= 252`（約 1 年）
     - 若不滿足則該股票不納入該產業樣本
5. 方法：策略定義與成本模型（需與現有 `backtest.py` 一致）
   - SMA 定義、訊號、`position = signal.shift(1)`、報酬 close-to-close
   - 成本：`cost_t = |Δsignal_{t-1}| * fee_bps / 10000`
   - 指標：Sharpe（rf=0, 252 日）、CAGR、MDD、年化波動等
6. 產業別參數最佳化（核心設計，避免 overfit）
   - 時間切分（決策完成）：
     - Train：`2010-01-01` ~ `2018-12-31`
     - Test：`2019-01-01` ~ `2025-12-31`
   - 參數格點（決策完成，放 Appendix 也要在本文提到）：
     - `short ∈ {5, 10, 20, 30, 40}`
     - `long  ∈ {20, 60, 120, 200}` 且 `long > short`
   - 每個產業的目標函數（決策完成）：
     - 對每組參數，先對該產業每檔股票算 Train 期策略日報酬
     - 計算每檔 Sharpe，取「產業內股票 Sharpe 的中位數」作為該組參數的產業分數
     - 選分數最高的 (short,long)；同分 tie-break：
       1) Train 期打敗 buy&hold 的股票比例較高者
       2) `long` 較大者（降低換手與過擬合風險）
7. 適用性定義（主結果指標）
   - OOS（Test）期對每檔股票計算：
     - `Outperform_i = 1` 若 `TotalReturn_strategy_i > TotalReturn_bh_i`，否則 0
   - 產業適用性指標（主表）：
     - `Applicability_industry = mean(Outperform_i)`（該產業打敗比例）
   - 次要指標（輔助表/附錄）：
     - 產業內股票「超額報酬」中位數、Sharpe 中位數、MDD 中位數
8. 統計檢定與多重比較（必寫，支撐“檢證”）
   - 產業內適用性比例的信賴區間：
     - 以股票為單位 bootstrap（對股票重抽樣 5,000 次）得到 95% CI
   - 檢定：
     - H1：`Applicability_industry > 0.5`（或 > 全市場基準比例）以 bootstrap p-value 或二項檢定（在論文中固定採 bootstrap）
     - 產業間差異：以比例差的 bootstrap 分布做兩兩比較（或以 logistic regression + 產業 dummy）
   - 多重比較校正：
     - 對所有產業的 p-value 做 Benjamini–Hochberg FDR（q=0.05），並在主表提供校正後顯著性標記
9. 結果呈現（表圖清單，論文內先放模板）
   - Table 1：樣本描述（各產業股票數、Train/Test 有效天數分布摘要）
   - Table 2：各產業最佳 (short,long)、Train 期產業分數（Sharpe 中位數）
   - Table 3（主表）：各產業 OOS 適用性比例 + 95% CI + FDR 後顯著性
   - Figure 1：適用性比例條狀圖（含 CI）
   - Figure 2：最佳化參數的分布（short/long 於產業上的散點或 heatmap）
10. 穩健性檢查（至少 3 組，寫死）
   - 成本敏感度：`fee_bps ∈ {0, 10, 20}`
   - 使用未調整價格表（`price_daily`）重跑對照
   - 子樣本：`2010-2014`、`2015-2019`、`2020-2025`（固定切段）檢查結論一致性
11. 討論、限制與結論
   - 可能機制：產業景氣循環、消息擴散、波動結構、趨勢延續性
   - 主要限制：產業分類變動、未納入退市股票（存活者偏誤）、成本模型簡化、無放空
   - 結論：哪些產業較適用、參數差異是否系統性、研究與實務含義
